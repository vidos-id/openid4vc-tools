## ADDED Requirements

### Requirement: Issuer CLI package exposes issuer flows through validated commands
The `issuer-cli` package MUST consume the `issuer` package and expose commands needed to run the supported demo issuance flow. Every command input MUST be validated with Zod schemas before the CLI calls issuer package APIs. The CLI MUST expose at least `metadata`, `create-offer`, `create-grant`, `nonce`, `issue`, and trust-material generation commands.

#### Scenario: Operator inspects issuer metadata
- **WHEN** a user runs the issuer CLI metadata command with valid issuer configuration input
- **THEN** the CLI validates the input with Zod and outputs issuer metadata generated by the `issuer` package

#### Scenario: Operator prepares a pre-authorized issuance flow
- **WHEN** a user runs the issuer CLI command to create a pre-authorized grant or offer
- **THEN** the CLI validates the command options with Zod and returns grant data created by the `issuer` package

### Requirement: Issuer CLI package supports claim-set driven issuance
The `issuer-cli` package MUST include a credential issuance command that accepts a claim set input and passes it to the `issuer` package for issuance. The CLI MUST validate the claim set structure and required issuance parameters before requesting issuance.

#### Scenario: Operator issues a credential from a claim file
- **WHEN** a user runs the issuer CLI issuance command with a claims JSON file, a `vct`, and holder binding inputs
- **THEN** the CLI validates the inputs and invokes the `issuer` package to issue a holder-bound `dc+sd-jwt` credential containing the provided claims

#### Scenario: Operator receives validation errors for malformed claims input
- **WHEN** a user runs the issuer CLI issuance command with malformed JSON or missing required issuance options
- **THEN** the CLI rejects the request before calling the `issuer` package and returns validation errors describing the invalid input

### Requirement: Issuer CLI package defines detailed command options
The `issuer-cli` package MUST provide detailed, documented command options for the supported demo flows. At minimum it MUST expose commands for metadata output, pre-authorized grant creation, and credential issuance.

#### Scenario: Metadata command options are available
- **WHEN** a user runs issuer CLI help for the metadata command
- **THEN** the CLI documents options including `--issuer`, `--config`, `--jwks` or equivalent signing-key source input, and `--output json|pretty`

#### Scenario: Grant command options are available
- **WHEN** a user runs issuer CLI help for the `create-grant` command
- **THEN** the CLI documents options including `--issuer`, `--credential-configuration-id` or `--vct`, `--subject-id` if supported, `--claims-file` if the grant prebinds claims, `--expires-in`, and `--output json|pretty`

#### Scenario: Nonce command options are available
- **WHEN** a user runs issuer CLI help for the `nonce` command
- **THEN** the CLI documents options including `--issuer`, `--config`, and `--output json|pretty`

#### Scenario: Issuance command options are available
- **WHEN** a user runs issuer CLI help for the issue command
- **THEN** the CLI documents options including `--issuer`, `--access-token` or grant-derived token input, `--credential-configuration-id` or `--vct`, `--claims-file`, `--proof`, `--proof-file`, `--signing-key`, and `--output json|pretty|raw`

#### Scenario: Trust material command options are available
- **WHEN** a user runs issuer CLI help for the trust-material generation command
- **THEN** the CLI documents options including issuer key output paths, certificate output paths, subject details if supported, and `--output json|pretty`
