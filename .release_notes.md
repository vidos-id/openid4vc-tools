# v0.7.0

## Highlights

- **Simplified wallet-cli output**: Commands now print concise, human-readable text by default while keeping `--json` and `--raw` modes for machine-oriented workflows
- **Automatic status resolution**: `wallet-cli show` resolves credential status automatically and warns instead of failing when status lookups cannot be completed
- **Persisted status list JWTs**: Issuer web server now stores status list JWTs between updates, improving reliability
- **Simplified CLI installation**: Streamlined README instructions for getting started with both CLIs

## New Features

- `wallet-cli` commands support `--format json` and `--format raw` output modes
- `wallet-cli show` automatically resolves credential status from the status list
- Status list JWTs are persisted in the database across updates in the issuer web server

## Other Changes

- Simplified CLI installation documentation in READMEs (root, issuer-cli, wallet-cli, pid example)
- Added database migration for status list JWT persistence (`0003_store_status_list_jwt.sql`)
