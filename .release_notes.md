## Highlights

- Refactored `issuer-cli` into a terminal app client for `issuer-web-server`, replacing local issuer-material management with server-backed auth, templates, issuances, and interactive flows.
- Added default interactive mode to both CLIs using `inquirer`, so running `issuer-cli` or `wallet-cli` without a subcommand opens a prompt-driven workflow.
- Made OID4VCI the primary credential-ingest path in `wallet-cli`, with `receive` promoted ahead of raw credential import.

## New Features

- Added interactive, menu-driven mode to `issuer-cli` for server sign-in, template management, and issuance management.
- Added interactive, menu-driven mode to `wallet-cli` for wallet setup, OID4VCI receipt, credential browsing, and presentation.
- Added server-backed session handling and formatted terminal output for `issuer-cli`.

## Other Changes

- Removed legacy local issuer and trust-material workflows from `issuer-cli`.
- Updated root and package READMEs to document the new server-backed issuer flow and wallet-first OID4VCI flow.
- Added `inquirer` to both CLI bundles to support richer interactive prompts.
